## 1. Безопасность памяти — стек (MIPS 8KB)

> Нарушение любого пункта = stack overflow на роутере = silent crash без backtrace.

- [ ] Ни один локальный буфер (`char buf[]`, массив на стеке) не превышает 512 байт ни в одной функции
- [ ] Нет `alloca()` нигде в кодовой базе (поиск: `grep -r alloca`)
- [ ] Нет VLA (variable-length arrays): `char buf[n]` где `n` — переменная
- [ ] Нет рекурсии без явного счётчика глубины с проверкой перед каждым вызовом
- [ ] Функции с большими локальными структурами переписаны на heap (`malloc` + `free`)
- [ ] `struct` размером > 256 байт не создаётся на стеке — только указатель
- [ ] Вложенные вызовы функций: суммарный стек-фрейм по критическому пути не превышает 4KB (оставить 4KB запас для прерываний MIPS)
- [ ] `splice` не используется нигде (data corruption на MIPS — запрещено навсегда)

## 2. Безопасность памяти — heap

- [ ] Каждый `malloc`/`calloc`/`realloc` проверяет результат на `!= NULL` до первого использования
- [ ] Каждый `malloc` имеет парный `free` на ВСЕХ путях: нормальном, ошибочном, и при `goto cleanup`
- [ ] После `free` указатель обнуляется: `ptr = NULL` — без исключений
- [ ] Нет double-free: проверяется `if (ptr != NULL)` перед `free` там, где возможен повторный вход
- [ ] `realloc` не теряет оригинальный указатель: `tmp = realloc(p, n); if (!tmp) { free(p); return ERR; } p = tmp;`
- [ ] Нет use-after-free: все структуры после `free` не адресуются ни через один alias
- [ ] **fake_ip OOM**: частичный rollback при OOM в `fake_ip` не оставляет stale pointer — полный rollback или fail-fast
- [ ] Структуры инициализируются через `= {0}` или `memset` до первого чтения любого поля
- [ ] Нет `memcpy`/`memmove` где `src` и `dst` перекрываются без `memmove`
- [ ] Все `strlen` вызываются только на гарантированно null-terminated строках

## 3. Безопасность памяти — строки и буферы

- [ ] `snprintf` везде вместо `sprintf` — без единого исключения
- [ ] Возвращаемое значение `snprintf` проверяется: `>= size` → truncation → ошибка
- [ ] Нет `strcpy`, `strcat` без явной проверки длины до вызова
- [ ] Нет `gets()` нигде
- [ ] `strncat(dst, src, n)` — `n` = `sizeof(dst) - strlen(dst) - 1` (не просто `sizeof(dst)`)
- [ ] `strncpy` всегда с явным `dst[n-1] = '\0'` после (null-терминатор не гарантирован)
- [ ] MAX_LINE = 8192 в `config.c` достаточен для `awg_i` hex blob — подтверждено реальным максимальным размером
- [ ] Все буферы для сетевых данных имеют explicit size check перед `memcpy`-подобными операциями
- [ ] Нет off-by-one: все индексы `[0..n-1]`, все срезы `[start, end)` — проверены на границах

## 4. Целочисленная арифметика

- [ ] Нет знаковых переполнений при индексации: все индексы `size_t` или `uint32_t`, не `int`
- [ ] Перед умножением проверяется переполнение: `if (a > SIZE_MAX / b) → ошибка`
- [ ] Нет сравнений `signed` vs `unsigned` без явного каста (флаг `-Wsign-compare` → 0 предупреждений)
- [ ] Нет битового сдвига на количество ≥ ширины типа (UB в C)
- [ ] Нет `int` для переменных хранящих размеры потенциально > 2GB — использовать `size_t`
- [ ] Все enum-значения используются только в своём диапазоне (нет `(enum_t)-1` без явного `INVALID` значения)

## 5. Корректность C-кода (gnu2x / musl)

### Обработка ошибок
- [ ] Каждый системный вызов (`read`, `write`, `close`, `open`, `socket`, `bind`, `connect`, `accept`) проверяет возвращаемое значение
- [ ] `write`/`send` в цикле — проверяется partial write (возвращаемое значение < запрошенного)
- [ ] `read`/`recv` проверяет `0` (EOF) и `-1` (ошибка) отдельно
- [ ] `close()` вызывается для всех fd на всех путях выполнения, включая error paths
- [ ] Нет игнорирования `close()` через `(void)close(fd)` — ошибка `close` на сетевых fd критична
- [ ] Все функции проекта возвращают коды ошибок и они проверяются caller'ом
- [ ] Нет `assert()` в production-коде как единственной проверки — только для invariants

### Стандарт gnu2x / musl
- [ ] Флаг `-std=gnu2x` (НЕ `-std=c2x`) во всех Makefile и сборочных командах
- [ ] `__VA_OPT__` используется только при `gnu2x` — проверено
- [ ] Нет glibc-специфичных функций: `error_t`, `program_invocation_name`, `mcheck`, `mtrace`
- [ ] `strerror_r` используется в XSI-совместимом варианте (musl реализует XSI, не GNU-расширение)
- [ ] Нет предположений о `errno` после функций не-libc (wolfSSL имеет свои коды ошибок)
- [ ] Нет POSIX-расширений недоступных в musl

### Управление потоком
- [ ] Все `switch` имеют `default` ветку
- [ ] Нет fall-through без явного `/* fallthrough */` или `__attribute__((fallthrough))`
- [ ] Нет недостижимого кода после `return`/`break`/`continue`
- [ ] Нет неинициализированных переменных: `= 0`, `= NULL`, `= {0}` при объявлении
- [ ] Цикломатическая сложность каждой функции ≤ 15 — иначе декомпозиция без обсуждений
- [ ] Функция не длиннее 80 строк — иначе декомпозиция

### Сигналы — критично
- [ ] `SIGHUP` → `handle_reload()` — обработчик установлен и корректен
- [ ] `SIGUSR1` — нет обработчика — нет ни одного вызова `kill(pid, SIGUSR1)` во всём проекте включая shell-скрипты
- [ ] `kill -HUP $(cat /var/run/4eburnet.pid)` — единственный способ reload в скриптах и документации

## 6. Безопасность (Security)

### IPC и shell injection
- [ ] IPC payload НИКОГДА не передаётся как shell-аргумент — только через tmp файл `/tmp/.4eburnet-ipc-PID.json`
- [ ] tmp файл создаётся с правами `0600` через `open(..., O_CREAT|O_EXCL, 0600)`
- [ ] tmp файл удаляется при любом выходе: нормальном, ошибочном, по сигналу
- [ ] PID в имени tmp файла исключает race condition (реальный `getpid()`, не `$$` в скриптах)
- [ ] Нет конкатенации UCI-значений напрямую в строку shell-команды
- [ ] Все `fs.popen()` вызовы в ucode: аргументы экранированы или передаются через безопасный механизм
- [ ] **SO_PEERCRED**: при ошибке `getsockopt(SO_PEERCRED)` — fail-secure (запретить, не разрешить)

### Входные данные
- [ ] Все поля UCI-конфига валидируются: тип, длина, диапазон — до первого использования
- [ ] Имена серверов/групп из UCI: проверка на shell-спецсимволы (`; & | $ \` ( ) { }`)
- [ ] Данные от upstream серверов: проверяется длина до парсинга
- [ ] POSIX ERE regex для filter/exclude: `regcomp()` результат проверяется, ошибка логируется и возвращается в caller
- [ ] AWG private key: никогда не попадает в лог ни на каком уровне (`LOG_DEBUG` включительно)
- [ ] Нет sensitive данных (ключи, пароли, SNI) в `/tmp` файлах после завершения операции

### wolfSSL
- [ ] Версия wolfSSL зафиксирована: `5.9.0` — не `latest`, не диапазон
- [ ] `wolfSSL_Init()` вызван ровно один раз при старте
- [ ] `wolfSSL_Cleanup()` вызван при завершении по всем путям
- [ ] Каждый `WOLFSSL_CTX*` имеет парный `wolfSSL_CTX_free()`
- [ ] Каждый `WOLFSSL*` имеет парный `wolfSSL_free()`
- [ ] Возвращаемое значение каждого TLS-вызова проверяется против `SSL_SUCCESS`
- [ ] `wolfSSL_get_error()` вызывается при любом non-SUCCESS коде для логирования
- [ ] Верификация peer certificate включена: `SSL_VERIFY_PEER` — не `SSL_VERIFY_NONE` в production
- [ ] TLS минимальная версия: TLS 1.2 (`wolfSSL_CTX_SetMinVersion(ctx, WOLFSSL_TLSV1_2)`)
- [ ] Нет хардкода cipher list без обоснования в комментарии

### Сеть
- [ ] DNS: нет DoH/DoT — запрещено архитектурно
- [ ] Нет unbounded `recv` с сокета без явного `maxlen`
- [ ] Таймаут установлен на каждую сетевую операцию: connect, read, write
- [ ] Rate limiting на входящих соединениях HTTP Dashboard :8080
- [ ] `write_all()`: перед отправкой — `fcntl(F_SETFL, ~O_NONBLOCK)` для блокирующего write — реализовано и не регрессировало
- [ ] EAGAIN fix для `logo.png` (176KB) — реализован и не регрессировал

### nftables
- [ ] `fwmark 0x01 → table 100`: добавляется через hotplug WAN, не только при старте демона
- [ ] Hotplug-скрипт идемпотентен: повторный вызов не создаёт дублирующих правил
- [ ] При `stop` IPC: все nftables правила и ip rules удаляются полностью
- [ ] fake_ip range `198.51.100.0/24` — RFC 5737 reserved, не публично маршрутизируемый
- [ ] TPROXY применяется только к TCP/UDP явно, не ко всем протоколам
- [ ] Нет fail-open: при отсутствии geo файлов поведение задокументировано и предсказуемо

## 7. RAM и производительность (MT7621A 116MB)

> При `available < 5MB` OOM убивает dropbear. Потеря SSH = физический ребут.

- [ ] Перед каждым действием на роутере в деплой-скриптах: `ssh root@192.168.2.1 "free -m"` — проверка
- [ ] `available < 15MB` → скрипт деплоя СТОП с явным предупреждением
- [ ] Демон RSS ~38MB — подтверждено, не растёт при нормальной работе (первые 24ч)
- [ ] 877K доменов в heap: нет дублирования строк, нет per-entry overhead > 32 байт
- [ ] `geo_loader.c`: нет повторной загрузки geo файлов без явной команды
- [ ] При SIGHUP (reload): geo данные не перегружаются полностью если reload не требует geo
- [ ] Нет malloc в hot path обработки DNS запроса/пакета — только pre-allocated структуры
- [ ] Нет утечки fd: все сокеты и файлы закрываются; `lsof -p <pid>` не растёт со временем
- [ ] Нет утечки памяти в обработчике DNS запросов (100K запросов → RSS не растёт)
- [ ] `geosite-ru.lst` (415KB), `geosite-ads.lst` (419KB) — streaming парсинг, не `fread` всего файла в буфер
- [ ] **v1.1 mmap placeholder**: TODO задокументирован в коде, не является активным обходом
