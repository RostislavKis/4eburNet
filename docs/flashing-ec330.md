# Прошивка OpenWrt на TP-Link EC330-G5u v1

## Информация об устройстве

| Параметр | Значение |
|----------|----------|
| Модель | TP-Link EC330-G5u (Archer C9ERT) Ver:1.0 |
| SoC | MediaTek MT7621 |
| Архитектура | mipsel_24kc |
| RAM | 128 МБ DDR2 |
| Flash | 32 МБ |
| Текущая ОС | KeeneticOS (ISP-прошивка) |
| Целевая ОС | OpenWrt 23.05.5 |
| Заводской IP | 192.168.0.1 |

## Файлы прошивки

- `openwrt-23.05.5-ramips-mt7621-tplink_ec330-g5u-v1-squashfs-factory.bin` — для прошивки из стокового веб-интерфейса
- `openwrt-23.05.5-ramips-mt7621-tplink_ec330-g5u-v1-squashfs-sysupgrade.bin` — для обновления уже работающего OpenWrt

Файлы лежат в: `~/phoenix-router-dev/firmware/`

## Подготовка

### 1. Резервная копия настроек KeeneticOS

1. Подключись к роутеру по LAN-кабелю
2. Открой http://192.168.1.1 (текущий IP, настроенный провайдером)
3. Войди: логин `admin`, пароль `admin`
4. Перейди: **Управление → Системные файлы** или **Общие настройки → Резервная копия**
5. Скачай файл конфигурации на компьютер
6. Запиши текущие настройки: IP WAN, DNS, Wi-Fi SSID/пароль

### 2. Физическая подготовка

- Подключи компьютер к LAN-порту роутера **напрямую** (не через другой роутер)
- Отключи Wi-Fi на компьютере, чтобы не было конфликта маршрутов
- Убедись, что Flint 2 (192.168.1.1) **не подключён** к этому же сегменту сети

## Прошивка из KeeneticOS

### Вариант A: Через веб-интерфейс KeeneticOS

1. Открой http://192.168.1.1, войди под admin/admin
2. Перейди: **Управление → Системные файлы**
3. Найди раздел «Обновление прошивки» или «Firmware»
4. Загрузи файл: `openwrt-23.05.5-ramips-mt7621-tplink_ec330-g5u-v1-squashfs-factory.bin`
5. Подтверди прошивку
6. **НЕ ОТКЛЮЧАЙ ПИТАНИЕ!** Процесс занимает 2-5 минут
7. Роутер перезагрузится с OpenWrt

### Вариант B: Через TFTP Recovery (если завис или не грузится)

Если роутер ушёл в кирпич или завис при прошивке:

1. Скачай и установи TFTP-сервер:
   - Windows: [Tftpd64](https://bitbucket.org/phjounin/tftpd64/downloads/)
   - Положи factory.bin в папку TFTP-сервера

2. Настрой статический IP на компьютере:
   ```
   IP: 192.168.0.66
   Маска: 255.255.255.0
   Шлюз: 192.168.0.1
   ```

3. Подключи LAN-кабель в порт LAN роутера

4. Зажми кнопку Reset на роутере

5. Включи питание роутера, **продолжая держать** Reset

6. Держи Reset 10 секунд, затем отпусти

7. Роутер войдёт в режим TFTP-восстановления и запросит файл прошивки

8. TFTP-сервер автоматически отправит файл

9. Дождись перезагрузки (~3-5 минут)

## Первичная настройка OpenWrt

После успешной прошивки:

### 1. Подключение

OpenWrt по умолчанию доступен на **192.168.1.1**.
Поскольку Flint 2 тоже на 192.168.1.1 — **отключи Flint 2 от сети** на время настройки.

1. Подключи LAN-кабель к EC330
2. Отключи Wi-Fi на компьютере
3. Открой http://192.168.1.1 — откроется LuCI (веб-интерфейс OpenWrt)

### 2. Установка пароля root

1. В LuCI: **System → Administration**
2. Задай пароль root
3. Или через SSH:
   ```sh
   ssh root@192.168.1.1
   passwd
   ```

### 3. Смена IP на 192.168.2.1

Чтобы EC330 не конфликтовал с Flint 2:

```sh
ssh root@192.168.1.1
uci set network.lan.ipaddr='192.168.2.1'
uci commit network
/etc/init.d/network restart
```

После этого роутер будет на **192.168.2.1**.
Переподключись: `ssh root@192.168.2.1`

### 4. Автоматическая настройка

Скопируй и запусти скрипт первичной настройки:

```sh
scp scripts/first-boot-ec330.sh root@192.168.2.1:/tmp/
ssh root@192.168.2.1 'sh /tmp/first-boot-ec330.sh'
```

### 5. Проверка

После настройки убедись:
- EC330 доступен на 192.168.2.1
- Flint 2 доступен на 192.168.1.1
- Оба роутера не конфликтуют
- SSH работает на EC330

## Откат на KeeneticOS

Если нужно вернуть стоковую прошивку:
1. Скачай оригинальную прошивку KeeneticOS с сайта провайдера
2. В OpenWrt LuCI: **System → Backup / Flash Firmware → Flash new firmware**
3. Или через TFTP Recovery (вариант B выше)

## Важные замечания

- Заводской IP этого роутера (по наклейке): 192.168.0.1
- Провайдер сменил IP на 192.168.1.1 и поставил KeeneticOS
- DNS-суффикс текущей сети: "Dlink" (настройка от провайдера)
- MAC: 88:FF:7B:2B:3C:52
- После прошивки OpenWrt factory-образ содержит только базовую систему
  без LuCI — может потребоваться установка через opkg
