# Архитектура phoenix-router

## Компоненты

### phoenix-core
Основной бинарник. Работает как демон, управляет:
- DNS-перехватом и классификацией доменов
- Правилами маршрутизации через nftables
- Прокси-соединениями (Shadowsocks, VLESS, Trojan)
- Мониторингом здоровья серверов

### luci-app-phoenix
Веб-интерфейс для настройки через LuCI.
Взаимодействует с phoenix-core через Unix-сокет (IPC).

### rules
Списки доменов и IP-адресов для выборочной маршрутизации.
Обновляются по расписанию через cron.

## Потоки данных

```
Клиент -> DNS-запрос -> resolver -> classifier -> ответ
                                       |
                                       v
Клиент -> TCP/UDP -> nftables -> tproxy -> dispatcher -> protocol handler -> сервер
```
